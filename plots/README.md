# Plots

Every figure here is generated by the current `analysis/` pipeline (see `../architecture.md`). Comparisons are between the multistream method, the 2D reference solve, and, where noted, the exact analytical solution.

## `convergence_at_t0.png`

The headline validation result. At t = 0, before any time evolution, the multistream moment matches the exact analytical moment to machine precision (about 1e-16) once M ≥ 3. This follows directly from the Gauss-Hermite quadrature being exact for polynomials up to degree 2M-1.

## `varying_streams/`

Fixed time, increasing number of streams M. Shows how quickly the multistream method converges toward the 2D reference solution, at five different points in the simulation:

- `T_0.1.png`, `T_0.5.png`, `T_1.png`, `T_2.png`, `T_5.png`

Across all five, accuracy improves fast up to about M = 3-4 streams, then plateaus, this is the computational saving the method is built on.

## `varying_time/`

Fixed number of streams, increasing simulation time. Shows how the error grows as the simulation runs longer.

- `all_M.png`, `all_M_logscale.png`: every tested M, linear and log scale
- `M_3_6_9.png`: the same comparison, restricted to M = 3, 6, and 9 for readability

Error growth is smooth and predictable up to about t = 1. Past that point the method's behaviour becomes less stable, an open limitation, discussed in `../architecture.md`.

## `spatial_profiles/`

The moment error plotted across physical space (not error magnitude vs. M or T, but where in space the error is largest), for a few different stream counts.

- `M_20.png`, `M_3_9_27.png`, `M_3_9_27_81.png`
